# Plan 04-01: Type Safety and Component Optimization (Updated)

Eliminate 'any' types, optimize hook dependencies, and strengthen UI component prop types with strict theme adherence.

## Tasks

### 1. Refactor useAsyncAction.ts (REQ-4.1)

- [ ] Change `Args extends any[]` to `Args extends unknown[]`.
- [ ] Destructure `onSuccess` and `onError` from `options`.
- [ ] Update `useCallback` dependency array to include `action`, `onSuccess`, and `onError`.

### 2. Strengthen Button Styles (REQ-4.2)

- [ ] In `components/ui/button.tsx`, replace `any[]` array styles with typed `StyleProp<ViewStyle>` or `StyleProp<TextStyle>`.

### 3. Strict Prop Types and Theme Resolution (REQ-4.3, REQ-4.4)

- [ ] In `components/ui/typography.tsx` and `components/ui/box.tsx`:
    - [ ] Refactor `resolveThemeColor` to remove `Record<string, any>` casting.
    - [ ] **Strict Typing**: Restrict `color`, `bg`, `borderColor` props to **only** valid theme keys.
    - [ ] **Strict Typing**: Restrict `p`, `m`, `gap`, etc., props to **only** valid `keyof typeof theme.spacing`.
    - [ ] Remove all raw number/string fallbacks for these props.

### 4. Optimize List Component and Callers (REQ-4.5)

- [ ] Update `ListProps<T>` in `components/ui/list.tsx` to make `keyExtractor` mandatory.
- [ ] Remove internal guessed `keyExtractor` logic from `FlatList`.
- [ ] Search the entire project for `<List` and update all callers with a specific `keyExtractor`.

## Verification Plan

### Automated Tests

- [ ] Run `npm run check:all` to ensure no TypeScript or Lint regressions.

### Manual Verification

- [ ] Verify `Button` and `Typography` components in various screens.
- [ ] Verify all `List` usages function correctly.
